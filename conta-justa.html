Aplicativo

Introdução

A empresa citada é a Justa Pagamentos, uma fintech sobre meios de pagamentos, focada em pequenos varejistas por todo o Brasil.

Problema

Atualmente os clientes varejistas usam o site (Portal do Lojista), para gerenciar as suas vendas e configurar os adiantamentos de recebíveis. De acordo com feedbacks recebidos pelos mesmos durante suas jornadas, o site precisaria de uma versão responsiva para o gerenciamento do negocio fluir de forma rápida e eficiente através do seu celular, já que o site não foi feito com um suporte/tecnologia para ser visualizado em um dispositivo menor.

Solução

Criar um aplicativo intuitivo e simples para ajudar o lojista a gerenciar as suas vendas de forma fácil e mais próxima com a sua realidade.

Desafio

O meu desafio no projeto como product designer foi:

Levantar todos os requisitos mínimos para uma primeira entrega, junto com o stake holder responsável pelo projeto.

Analisar junto com o Product Owner os endpoints já criados anteriormente no portal WEB do lojista, para implementar os mesmos no APP.

Realizar uma análise de competitividade para entender fluxos desejados já existentes no mercado.

Criar wireframe de alta fidelidade para validação interna de novas hipóteses.

Criar protótipo navegável para testes internos, e também para auxiliar a equipe responsável no desenvolvimento do projeto.

Realizar "Design Review" junto com o time durante a Sprint. Garantindo uma experiência / interface mais próxima do planejado.

Trazer uma solução consistente para analise de desempenho do APP, seja elas na parte técnica e também na experiência durante as jornadas.

Levantar insights do APP através de pesquisas com usuários ativos.

Análises

Para testar o desenvolvimento das hipóteses criadas na primeira versão, implementamos os seguintes processos:

01

SDK do Google (FIREBASE)

Criamos eventos durante jornadas especificas dentro do APP com o objetivo de mensurar a porcentagem de desistências dos usuários.

02

Feedback induzido

Adicionamos um modal que aparece uma vez a cada 3 dias pedindo para o usuário avaliar o APP na loja. Caso o usuário já tenha clicado em avaliar dentro do modal, ele passa a aparecer 1 vez por mês (Período médio de uma nova release).

03

Pesquisa NPS

Rodamos uma pesquisa NPS no canal de comunicação da empresa, para saber o nível de desempenho do APP e o quanto o usuário recomendaria para as outras pessoas.

04

SDK da Microsoft (AppCenter)

Implementamos um SDK mais especializado nas análises técnicas do APP, para acompanhar o desempenho das API's através dos disparos de erros e crashes.
